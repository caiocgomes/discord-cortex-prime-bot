# Cortex Prime: Regras Mecânicas Extraídas do Primer

Fonte: Cortex Prime Game Handbook, Capítulo "Read This First: The Cortex Primer" (pp. 6-13) complementado pelo Glossário (pp. 228-237).

---

## Dados e Ratings

Cortex Prime usa cinco tipos de dados: d4, d6, d8, d10 e d12. Não usa d20. Cada trait de um personagem tem um die rating associado, que pode ser um tipo de dado (ex: d8), múltiplos dados do mesmo tipo (ex: d8d8) ou combinação de tipos (ex: d4 + d12). A escala de poder segue a progressão d4 < d6 < d8 < d10 < d12 (p. 8, 16).

## Traits e Trait Sets

Toda mecânica de personagem gira em torno de traits: um rótulo descritivo com um die rating. Os traits são organizados em trait sets (categorias como atributos, skills, distinctions, powers, values, affiliations, roles, relationships, resources). Cada jogo Cortex Prime escolhe quais trait sets usa. Os trait sets designados como prime sets são a fonte essencial de dados para os dice pools (p. 8, 26, 46-47).

Ao montar um dice pool, o jogador pega um dado de cada trait set relevante do seu character file. Isso significa que o pool é construído aditivamente por camadas: um dado do atributo, um dado da skill, um dado da distinction, etc., conforme os sets escolhidos para o jogo (p. 8-9).

## Dice Pool: Montagem e Resolução

### Montagem do Pool

O jogador seleciona um dado de cada trait set aplicável à ação. Além disso, o pool pode receber dados adicionais de assets, complications (no pool da oposição), plot points gastos, e efeitos especiais (SFX) (p. 8-9, 17).

Na versão simplificada do primer: se a trait central do personagem se aplica, rola d6d6d6 (três d6); se não se aplica, rola apenas d6d6 (dois d6) (p. 6-7).

### Roll and Keep

Independente de quantos dados são rolados, o jogador por default mantém apenas dois dados para somar como total. Os demais dados rolados ficam disponíveis como possíveis effect dice. Quanto mais dados e maiores eles forem, melhor a chance de um total alto, mas também maior a chance de hitches (p. 8, 17).

### Total

O total é a soma dos dois dados mantidos (os dois maiores, por default). Esse número é comparado contra o total da oposição para determinar sucesso ou falha (p. 8, 16-17).

### Effect Die

Depois de escolher os dois dados para o total, o jogador escolhe um dos dados restantes como effect die. Geralmente pega-se o maior dado remanescente. O effect die serve para determinar a magnitude do efeito: criar assets, infligir complications, definir a escala do sucesso. Em caso de vitória, o effect die define o tamanho do asset criado ou da complication infligida (p. 20, 159, 237-239).

## Dificuldade e Oposição

### Tests

Um test ocorre quando o personagem quer fazer algo e existe algo que pode impedir. O GM monta um pool de oposição e rola para estabelecer a dificuldade. O jogador precisa superar esse total para ter sucesso. Se o total do jogador for igual ou menor que a dificuldade, falha (p. 6-7, 18).

Na versão simplificada do primer:
- Oposição fácil: GM rola d6 (p. 6)
- Oposição desafiadora: GM rola d6d6 (p. 6)
- Oposição difícil: GM rola d6d6d6 ou mais (p. 6)

### Contests

Um contest é uma série de rolagens opostas entre dois lados. Cada lado tenta superar a rolagem anterior do oponente. Continua até um lado escolher dar in (give in) ou falhar em superar o total anterior. Quem falha ou dá in sofre uma complication ou é taken out. Mais de dois participantes podem entrar num contest (p. 19, 23).

### Heroic Success

Se o total do jogador supera a dificuldade por 5 ou mais pontos, é um heroic success. Isso pode gerar hero dice (mod) ou efeitos ampliados (p. 20, 30-31).

### Botch

Se todos os dados do jogador saem como 1 (hitches), o total é zero. Isso é um botch. O GM pode introduzir uma complication d6 gratuita e step up essa complication uma vez para cada hitch além do primeiro. Nenhum plot point é dado ao jogador num botch (p. 17, 19, 28).

## Hitches e Opportunities

### Hitch (Jogador rola 1)

Qualquer dado que resulta em 1 é um hitch. Esse dado não conta para o total. O GM pode dar ao jogador um plot point (PP) para transformar o hitch em uma complication. Hitches adicionais na mesma rolagem fazem step up da complication (p. 7, 17, 28).

### Opportunity (GM rola 1)

Quando o GM rola um 1, isso é uma opportunity. O jogador pode gastar um PP para step down uma complication existente ou step up um asset existente. Múltiplas opportunities na mesma rolagem permitem múltiplos steps (p. 37, 489-494 do glossário).

## Plot Points (PP)

### Economia Básica

Todo personagem começa com pelo menos um plot point. Plot points ficam num bank individual do jogador. Existe também um pile central de onde os PPs vêm e para onde voltam (p. 6, 27).

### Ganhar PPs

- Quando o jogador rola um hitch e o GM opta por ativar a complication, o jogador recebe um PP (p. 7, 28).
- Dar in num contest sem rolar gera um PP (p. 19, 28).
- Outros métodos dependem dos mods escolhidos (distinctions usadas como d4, challenges, etc.) (p. 28).

### Gastar PPs

- Adicionar um d6 ao dice pool antes de rolar, justificando narrativamente (p. 7, 28).
- Ativar certos SFX (p. 28, 61-63).
- Criar um asset d6 (p. 28, 34-35).
- Interferir num contest já em andamento (custa 1 PP para entrar) (p. 23, 98).
- Usar uma opportunity da oposição para step down complication ou step up asset (p. 37).
- Evitar ser taken out ao aceitar uma complication d6 no lugar (p. 28-29, 37).

## Stepping Up e Stepping Down

Step up significa trocar um dado pelo próximo tamanho acima na escala: d4 vira d6, d6 vira d8, d8 vira d10, d10 vira d12. Step down é o inverso. Isso substitui modificadores flat, preservando a variância natural do dice pool (p. 9, 16).

Se um dado faz step down abaixo de d4, ele é eliminado (p. 38). Se uma complication faz step up além de d12, o personagem é taken out (p. 22, 37).

## Assets

Um asset é um trait temporário criado durante o jogo que é incluído no dice pool do jogador quando relevante. Pode ser criado gastando um PP (começa como d6) ou como resultado de um effect die numa rolagem bem-sucedida (nesse caso, o tamanho do asset é o tamanho do effect die escolhido). Assets representam vantagens circunstanciais, objetos, posicionamento, preparação, etc. (p. 9, 34-35).

Quando o GM rola uma opportunity (um 1), o jogador pode gastar um PP para step up um asset existente (p. 37).

Stunts são assets criados por SFX ou regras especiais, começando em d8 (p. 34).

Signature assets são assets permanentes que pertencem ao personagem e podem ser incluídos no pool quando aplicáveis (p. 64-65).

## Complications

Uma complication é um trait temporário negativo. Quando um personagem tem uma complication, a oposição pode incluir o dado da complication no seu dice pool, tornando a ação do personagem mais difícil (p. 9, 28, 34, 36-37).

Complications são criadas quando o GM ativa um hitch do jogador (d6 inicialmente, stepped up por hitches adicionais), ou quando a oposição vence e usa seu effect die para infligir uma complication (p. 36-37).

Se uma complication faz step up além de d12, o personagem é taken out. Como alternativa a ser taken out, o jogador pode aceitar uma complication d6 extra gastando um PP (p. 22, 37).

Um jogador pode gastar um PP quando a oposição rola uma opportunity para step down uma complication existente. Se a complication faz step down abaixo de d4, ela é eliminada (p. 37-38).

### Stress (mod de complication)

Stress é um tipo específico de complication que representa dano. Pode ter categorias (Physical, Mental, Emotional). Recovery de stress requer um teste contra uma dificuldade, e o tipo de stress determina as condições de recuperação (p. 39-40).

Trauma é stress de longo prazo, mais difícil de recuperar, ganho quando um personagem é "stressed out" (p. 41).

## Distinctions

Distinctions são traits que representam background, personalidade e papel narrativo do personagem. São universais (todo jogo Cortex Prime as usa). Normalmente rated em d8. O jogador pode optar por rolar a distinction como d4 em vez de d8, ganhando um PP em troca. Isso representa a distinction atrapalhando em vez de ajudando (p. 50, 61).

## SFX (Special Effects)

SFX são regras especiais ligadas a traits (geralmente powers, distinctions ou abilities). Cada SFX tem um cost (o que ativa, ex: gastar um PP, step down um dado) e um benefit (o que ganha, ex: adicionar dado extra, step up effect die, reroll). SFX podem ser shut down por limits, que desativam o power set inteiro em troca de um PP (p. 61-63, 193).

## Mods Relevantes para Implementação

### Doom Pool (mod que substitui PPs do GM e dificuldade)

O doom pool é um conjunto de dados do GM que substitui a necessidade de difficulty dice e PPs do GM. O GM gasta dados do doom pool para criar oposição, ativar efeitos, e pode adicionar dados ao doom pool a partir de hitches dos jogadores. O doom pool cresce e encolhe durante o jogo (p. 32-33).

### Hero Dice (mod)

Hero dice são dados ganhos por heroic successes que ficam reservados para rolagens futuras. Podem ser usados em vez de (ou junto com) dados normais do pool. Com static difficulty, hero dice podem substituir um dado na dificuldade fixa (p. 25, 30-31).

### Action Order (mod de conflito)

Substitui tests/contests por uma mecânica de turnos. Personagens agem em ordem, rolam primeiro (action) e depois o alvo reage (reaction). A ordem pode ser determinada por iniciativa ou dramatic order (p. 24, 98-99).

### Give In

Em qualquer momento num contest, antes de rolar, o jogador pode escolher dar in. Isso evita ser taken out e dá ao jogador algum controle narrativo sobre as consequências. Dar in pode gerar um PP dependendo do mod (p. 19, 28).

## Resumo da Sequência de Resolução (para implementação no bot)

1. **Montar dice pool**: um dado de cada trait set relevante + assets + dados de PP gastos + SFX.
2. **Montar pool de oposição**: dados do GM (difficulty, doom pool, ou GMC traits) + complications do alvo.
3. **Rolar ambos os pools**.
4. **Identificar hitches**: dados que saem 1. Não contam para total. GM pode dar PP ao jogador e criar/step up complication.
5. **Escolher dois dados para total** (os dois maiores, por default).
6. **Escolher effect die** dos dados restantes (geralmente o maior).
7. **Comparar totais**: jogador > oposição = sucesso; jogador <= oposição = falha.
8. **Heroic success**: se total do jogador supera oposição por 5+.
9. **Aplicar effect die**: criar asset, infligir complication, ou outro efeito mecânico.
10. **Checar taken out**: complication stepped up além de d12 = taken out. Pode gastar PP para evitar.
